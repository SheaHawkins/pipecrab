Writing a Stage

A stage is two halves. The synchronous Processor half owns all state mutation and cannot be cancelled; the async Stage half does the I/O and can be dropped mid-flight — which is exactly what makes barge-in safe.

Decide: the synchronous half

A stage implements Processor. Both decide_data and decide_system return a Decision, which answers two questions at once: does the incoming frame keep moving downstream? and what should this stage emit?

You returnInput frameEmits
Decision::forward()forwarded downstreamnothing
Decision::drop()consumednothing
Decision::drop().emit(x)consumedx
Decision::forward().emit(x)forwarded downstreamx

Those four forms map onto three kinds of stage:

  • Transform (STT, a redactor): drop().emit(output) — the input never reaches downstream, only the replacement does.
  • Tap (VAD, a logger): forward().emit(derived) — the original passes through, followed by the derived frame.
  • Pass-through: don't override decide_data/decide_system — the default is Decision::forward(), so every frame on an ignored lane flows on unchanged.
rust 
fn decide_data(&mut self, frame: &DataFrame) -> Decision<Self::Effect> {
    match frame {
        DataFrame::Audio(a) => Decision::drop().emit(Effect::Transcript(self.stt(a))),
        _ => Decision::forward(),
    }
}

Because every arriving frame must be answered with a Decision, forwarding versus dropping is explicit and compiler-enforced — you can't accidentally forget to push a frame the way an imperative push_frame API lets you.

Effects: the stage's command vocabulary

Processor::Effect is plain data the stage defines for itself — commands describing what should happen. The decide half emits them; the runtime hands each one to perform, which interprets it and does the actual I/O.

decide_* must not do I/O. The one carve-out is a control call: a synchronous, non-blocking, idempotent, infallible operation on an owned engine — canonically cancel(), which flips an atomic flag a worker observes. Anything that can block, fail, or allocate unboundedly stays an effect for perform.

Perform: the async half

Stage extends Processor with the async interpreter:

rust 
async fn perform(&self, effect: Self::Effect, out: &Outbound) -> Result<(), StageError>;

Note the receiver: &self, not &mut self. perform cannot mutate stage state. The run loop races this future against the system lane, so a barge-in Interrupt can drop it mid-flight — and because only decide_* ever mutated state, dropping the future leaves the stage intact. This split is the whole trick: state mutation is uncancellable, cancellable work is stateless.

A complete stage

From the e2e voice agent: a gate that prints each completed user utterance and drops empty finals so a noise trigger never wakes the language model. Picking the text out is state-side (decide_data); printing is I/O (perform).

rust 
use pipecrab::{
    DataFrame, Decision, Finality, Outbound, Processor, Role, Stage,
    StageError, Transcript,
};

struct UserTurnGate;

/// One completed user utterance to print.
struct PrintUser(Arc<str>);

impl Processor for UserTurnGate {
    type Effect = PrintUser;

    fn decide_data(&mut self, frame: &DataFrame) -> Decision<PrintUser> {
        match frame {
            DataFrame::Transcript(Transcript {
                role: Role::User,
                finality: Finality::Final,
                text,
            }) => match text.trim().is_empty() {
                // A noise trigger scored as blank: consume it so the
                // LM does not generate a reply to an empty turn.
                true => Decision::drop(),
                false => Decision::forward().emit(PrintUser(text.clone())),
            },
            _ => Decision::forward(),
        }
    }
}

#[cfg_attr(target_arch = "wasm32", async_trait(?Send))]
#[cfg_attr(not(target_arch = "wasm32"), async_trait)]
impl Stage for UserTurnGate {
    async fn perform(
        &self,
        PrintUser(text): PrintUser,
        _out: &Outbound,
    ) -> Result<(), StageError> {
        println!("You: {text}");
        Ok(())
    }
}

Drop it into a pipeline like any built-in stage: .stage(UserTurnGate).

Off-thread work

The pipeline is one !Send thread, and perform must keep yielding so an interrupt can preempt it. Heavy work never runs inline: offload(f).await crosses to a std::thread on native, or a Web Worker on wasm. It is the only API with a Send bound — that bound is the thread-crossing boundary, not the pipeline.

rust 
async fn perform(&self, effect: Self::Effect, out: &Outbound) -> Result<(), StageError> {
    let engine = self.engine.clone();
    let text = offload(move || engine.transcribe(effect.audio)).await?;
    out.send_data(DataFrame::Transcript(text)).await?;
    Ok(())
}

System frames

decide_system receives Start, Stop, Interrupt, and Error along with the direction they're travelling, and defaults to forwarding them. The runtime handles the hard parts for you: system frames are drained before data, an Interrupt drops any in-flight perform, and the queued data backlog is flushed — transport-audio survivors excepted, so a barge-in utterance is never clipped.
