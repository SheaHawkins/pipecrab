API Reference

Full rustdoc for every published crate lives on docs.rs. Most applications depend on the pipecrab facade plus one adapter crate per capability; the trait crates matter when you're implementing an engine of your own.

Start here

CrateWhat it is
pipecrab The facade — re-exports core + runtime. PipelineBuilder, Processor, Decision, Stage, frames. docs.rs

Engine layers

Dependencies point strictly downward — core < runtime < traits < adapters — enforced by a workspace layering test.

CrateWhat it is
pipecrab-core Sans-IO: frames, Processor, Decision. No async, no I/O, zero deps. docs.rs
pipecrab-runtime Async orchestration: Stage, Pipeline, Inbound/Outbound, offload. No executor baked in. docs.rs

Capability traits

Each capability crate defines the swappable trait plus the stage that adapts an implementation into the pipeline.

CrateWhat it is
pipecrab-audio AudioSource/AudioSink, hardware-free mocks, and the streaming ResamplerStage. docs.rs
pipecrab-vad Two-tier VAD: VoiceActivityDetector + SpeechScorer, the Debounced lifter, and the VadStage gate. docs.rs
pipecrab-stt Transcriber trait + SttStage: replaces Audio with Transcript. docs.rs
pipecrab-lm Language-model capability + LmStage: user transcripts in, streaming agent replies out. docs.rs
pipecrab-tts Synthesizer trait, TtsStage, and the SentenceChunker that enables sentence-by-sentence speech. docs.rs

Adapters

Adapters bridge an external engine or device to a capability trait. Your app depends on these; the pipeline never names a concrete model.

CrateBridges
pipecrab-audio-cpal cpal microphones and speakers via a lock-free real-time-safe ring bridge. docs.rs
pipecrab-vad-sherpa sherpa-onnx Silero VAD → VoiceActivityDetector. docs.rs
pipecrab-stt-sherpa sherpa-onnx streaming Zipformer and offline Moonshine v2 → Transcriber. docs.rs
pipecrab-lm-llamacpp llama.cpp GGUF chat models → the LM capability. docs.rs
pipecrab-tts-sherpa sherpa-onnx Kokoro TTS → Synthesizer. docs.rs
