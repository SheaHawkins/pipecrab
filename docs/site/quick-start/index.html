Quick Start

Talk to a fully local voice agent in minutes. Sherpa VAD brackets each utterance from your microphone, Moonshine v2 transcribes it, a llama.cpp chat model streams a reply, and Kokoro speaks it back — all on your machine, no network calls.

Use headphones. Over speakers the microphone re-captures the agent's own voice, the VAD opens, and the agent starts talking to itself.

What you'll run

The e2e-voice-agent example is the full duplex loop, expressed as one pipeline:

pipeline 
CpalSource (mic)
    │ device sample rate, mono
    ▼
ResamplerStage (16 kHz mono)
    ▼
VadStage<SherpaVad>          gates: emits only utterances
    ▼
SttStage<OfflineSherpaStt>   Audio → user Transcript
    ▼
LmStage<LlamaCpp>            streams agent partials + one final
    ▼
SentenceChunker              one final transcript per sentence
    ▼
TtsStage<SherpaTts>          24 kHz mono audio per sentence
    ▼
ResamplerStage (device rate)
    ▼
CpalSink (speaker)

The SentenceChunker is why the agent starts talking before the model has finished writing: each completed sentence of the streaming generation is synthesized while the next sentence is still being generated.

Requirements

  • Rust 1.86 or newer.
  • macOS, Windows, or Linux with a working microphone and output device.
  • Microphone permission for the terminal running Cargo.
  • ~1.5 GB of disk for the four model downloads below.

1. Clone the repository

console 
$ git clone https://github.com/SheaHawkins/pipecrab.git
$ cd pipecrab

2. Download the models

Four local models power the loop: Silero VAD, Moonshine v2 (STT), a small Qwen chat model (LM), and Kokoro (TTS). From the repository root:

console 
$ mkdir -p models

# VAD — Silero
$ curl -L https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/silero_vad.onnx \
    -o models/silero_vad.onnx

# STT — Moonshine v2 (quantized)
$ curl -L https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/sherpa-onnx-moonshine-base-en-quantized-2026-02-27.tar.bz2 \
    -o models/sherpa-onnx-moonshine-base-en-quantized-2026-02-27.tar.bz2
$ tar xvf models/sherpa-onnx-moonshine-base-en-quantized-2026-02-27.tar.bz2 -C models

# LM — Qwen 2.5 0.5B instruct, GGUF
$ curl -L https://huggingface.co/Qwen/Qwen2.5-0.5B-Instruct-GGUF/resolve/main/qwen2.5-0.5b-instruct-q4_k_m.gguf \
    -o models/qwen2.5-0.5b-instruct-q4_k_m.gguf

# TTS — Kokoro en v0.19 (eleven speakers)
$ curl -L https://github.com/k2-fsa/sherpa-onnx/releases/download/tts-models/kokoro-en-v0_19.tar.bz2 \
    -o models/kokoro-en-v0_19.tar.bz2
$ tar xvf models/kokoro-en-v0_19.tar.bz2 -C models

The repository ignores models/, so the downloads never show up as source changes.

3. Run it

console 
$ ASR=./models/sherpa-onnx-moonshine-base-en-quantized-2026-02-27
$ TTS=./models/kokoro-en-v0_19

$ cargo run -p e2e-voice-agent -- \
    --vad-model ./models/silero_vad.onnx \
    --encoder "$ASR/encoder_model.ort" \
    --merged-decoder "$ASR/decoder_model_merged.ort" \
    --tokens "$ASR/tokens.txt" \
    --lm-model ./models/qwen2.5-0.5b-instruct-q4_k_m.gguf \
    --tts-model "$TTS/model.onnx" \
    --tts-voices "$TTS/voices.bin" \
    --tts-tokens "$TTS/tokens.txt" \
    --tts-data-dir "$TTS/espeak-ng-data"

Then speak. Expected output resembles:

output 
e2e-voice-agent: input = MacBook Pro Microphone @ 48000 Hz mono
e2e-voice-agent: output = MacBook Pro Speakers @ 48000 Hz
e2e-voice-agent: kokoro @ 24000 Hz mono, speaker 0, speed 1
e2e-voice-agent: listening until Ctrl-C
SpeechStarted
SpeechStopped (1.92 s)
You: what is a crab
Agent: A crab is a crustacean with a hard shell and ten legs.

…followed by the same sentence spoken aloud.

Useful flags

  • --speaker 8 — pick another Kokoro voice (0–10).
  • --speed 1.2 — talk faster.
  • --system-prompt "…" — change the agent's instructions.
  • --seconds 30 — bound the run.
  • --stt-threads 1 — low-power profile.

The first Sherpa build may download matching native libraries; set SHERPA_ONNX_LIB_DIR to reuse an existing Sherpa installation instead.
