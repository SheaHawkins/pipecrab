Build voice agents, in Rust.

pipecrab is a cross-platform pipeline framework for building fully local, interruptible voice agents.

Get started View on GitHub

cargo add pipecrab

rust 
let (ends, driver) = PipelineBuilder::new()
    .stage(ResamplerStage::new(FORMAT)?)  // → 16 kHz mono
    .stage(VadStage::with_config(vad, cfg)) // gate on speech
    .stage(SttStage::new(transcriber))      // audio → text
    .stage(LmStage::new(model, prompt))     // text → reply
    .stage(TtsStage::new(synthesizer))      // reply → audio
    .build()
    .start();

// No executor baked in — you drive the future.
block_on(async { futures::join!(driver, pump_in, drain) });

Designed for barge-in

A voice agent must stop talking the instant you speak. pipecrab's architecture makes interruption a first-class, state-safe operation instead of an afterthought.

◆Two lanes

Every pipeline link is a data lane plus a high-priority system lane. Interrupt, Start/Stop, and Error overtake queued work and can travel upstream; data flows strictly downstream.

◆Interrupt-safe by construction

State mutates only in the synchronous, uncancellable decide step. The async perform step does the I/O and can be dropped mid-flight without tearing state.

◆Compiler-enforced flow

Every stage returns a Decision that says whether the frame forwards or drops, and what it emits. You can't accidentally forget to push a frame — the type system won't let you.

◆Runs everywhere

No executor is baked in and every async trait is wasm-aware, so one stage definition runs under block_on natively and spawn_local in the browser. CI compiles the core for wasm32.

◆Local inference

Adapter crates bridge sherpa-onnx (VAD, STT, Kokoro TTS), llama.cpp, and cpal behind clean capability traits. The pipeline never names a concrete model.

◆Layered and enforced

core < runtime < traits < adapters — the dependency direction is an invariant checked by a workspace test, not a convention. Swap engines without touching your pipeline.
